# Структура CSS в TaskForge

## Обзор
В проекте TaskForge используется компонентный подход к организации CSS. Это позволяет улучшить поддерживаемость кода, упростить разработку и обеспечить лучшую масштабируемость.

## Структура директорий

```
/main/static/css/
  ├── main.css               # Главный CSS файл, импортирует все остальные
  ├── base/                  # Базовые стили
  │   ├── reset.css          # Сброс стилей браузера
  │   ├── typography.css     # Шрифты и типографика
  │   ├── colors.css         # CSS переменные с цветами
  │   └── global.css         # Глобальные стили
  │
  ├── components/            # Стили компонентов интерфейса
  │   ├── navbar.css         # Навигационная панель
  │   ├── buttons.css        # Кнопки
  │   ├── cards.css          # Карточки (привычки, цели)
  │   ├── calendar.css       # Календарь
  │   ├── progress.css       # Элементы прогресса (круги, полосы)
  │   ├── modals.css         # Модальные окна
  │   └── forms.css          # Формы и элементы ввода
  │
  └── layout/                # Макеты и структура страниц
      ├── dashboard.css      # Главная страница дашборда
      ├── stats-dashboard.css # Блок статистики
      └── footer.css         # Подвал сайта
```

## Принципы организации

1. **Базовые стили** содержат:
   - CSS-переменные для цветов, размеров и отступов
   - Сброс стандартных стилей браузера
   - Типографику (шрифты, размеры, заголовки)
   - Глобальные стили (фон, основные контейнеры)

2. **Компоненты** содержат:
   - Изолированные стили для отдельных компонентов интерфейса
   - Каждый компонент имеет свой файл CSS
   - Компоненты используют переменные из базовых стилей

3. **Макеты** содержат:
   - Стили для структуры страниц и секций
   - Расположение компонентов относительно друг друга
   - Медиа-запросы для адаптивности всего макета

## Использование

Для использования CSS в проекте достаточно подключить только файл `main.css`:

```html
<link rel="stylesheet" href="{% static 'css/main.css' %}" />
```

Все остальные файлы будут импортированы автоматически с помощью директивы `@import` в правильном порядке.

## Правила при разработке

1. **Новые компоненты**:
   - Создавайте отдельный файл в директории components/
   - Добавляйте импорт в main.css

2. **Изменения существующих стилей**:
   - Вносите изменения в соответствующий файл компонента или макета
   - Не изменяйте непосредственно файл style.css

3. **Использование переменных**:
   - Всегда используйте CSS-переменные для цветов и размеров
   - При необходимости добавляйте новые переменные в colors.css

## Преимущества

- Улучшенная поддерживаемость: легче находить и исправлять стили
- Предотвращение конфликтов имен: стили разделены по файлам
- Улучшенная читаемость: каждый файл имеет конкретную цель
- Масштабируемость: легко добавлять новые стили и компоненты
